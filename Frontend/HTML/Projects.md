## üß† Best Practices for All Projects

- **Semantic HTML**: `<header>`, `<footer>`, `<main>` over `<div>`  
- **Accessibility**: Labels, keyboard navigation, ARIA  
- **SEO**: Titles, meta descriptions, alt attributes  
- **Performance**: Minimize DOM, lazy-load media, optimize images  
- **Clean Code**: Proper indentation, comments, readable markup  
- **Mobile First**: Test on multiple devices

## üìå Extra Tips

- Always start with a proper **HTML Boilerplate**  
- Keep **class & id naming consistent** for CSS/JS integration  
- Experiment with **HTML entities**, URL encoding, and character sets  
- Learn **HTML vs XHTML differences** (self-closing tags, strict parsing)  
- Include structured data for blog / article projects  
- Use **HTML APIs**: `data-*`, `<template>`, `contenteditable`  
- Prepare HTML for JS frameworks & React components
